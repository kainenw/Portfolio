This is a music theory project that allows users to experiment with different kinds of chord changes. The chord changes are visualized on a grid, the tonnetz, which is a way to graphically visualize chords. There are buttons for each unique chord transformation that is possible within a neo-riemannean cclassification system. It is designed with HTML, CSS, and Javascript. It uses the html canvas, which was a small part of the Tonnetz-Viz repo, which I have adapted to my own purposes here. I have plans to allow the user to interact directly with the grid by clicking on it. I will implement this with the html canvas framework Konva.js.

This is a project that I started in order to generate lines of counterpoint according to the rules of Johann Fux. I have started with Fux's rules because they are the simplest and most clearly defined. At first, I programmed it to find the one possible counter-melody with the fewest leaps. After looking at some other projects with the same goal, I realized I wanted to get every solution and pick between them. I implemented this in a wildly ineficcient way, using lists which had to be copied and split for each note in the melody. This led me to rethink how to keep track of the different valid solutions. I am currently restructuring the data so that the different possibilies can be represented in a graph, with each node on the graph corresponding to one possibility for the counterpoint.
